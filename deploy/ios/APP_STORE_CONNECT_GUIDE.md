## üî¥ SE VIENI RIFIUTATO

**Don't panic!** Rifiuti sono comuni per prima submission.

### Motivi Comuni
1. **Guideline 2.1** (Crash/Bug): App crasha su reviewer device
   - Fix: Debug crash, update build, resubmit
   
2. **Guideline 4.0** (Design): UI non conforme
   - Fix: Improve UI, rispetta Human Interface Guidelines

3. **Guideline 5.1.1** (Privacy): Privacy policy mancante/incompleta
   - Fix: Verifica URL privacy policy funzioni

4. **Guideline 2.3.3** (Metadata): Screenshot/description non accurate
   - Fix: Update screenshots per riflettere app reale

### Rispondere a Rejection

1. Leggi **Resolution Center** message (dettagli rejection)
2. Fixa issue
3. **Update build** se necessario: `eas build --platform ios --profile production`
4. Update metadata se serve
5. **Reply in Resolution Center** spiegando fix
6. Click **"Submit for Review"** again
